# Sample Daily Report

This document explains the sections in a mock-mode sample report generated by `app-store-monitor daily --mock --print`. It does not contain real App Store Connect data, real app identifiers, private credentials, or production KPI history.

## Running the Report

```bash
python -m src.cli daily --mock --print
```

The mock flow initializes a local SQLite database, inserts deterministic sample metrics, writes a Markdown report, and prints it to stdout.

---

## Section-by-Section Guide

### Summary

The summary block gives a one-glance overview of key metrics across all tracked apps.

| Metric | Meaning |
|--------|---------|
| Apps with downloads: X / Y | How many apps recorded at least one download, out of the total number of apps being tracked. |
| Total downloads | Sum of all downloads across all apps for the reporting period. |
| Best CTR app | App with the highest Conversion Rate (CVR): downloads ÷ product page views. |
| Highest impressions app | App that appeared in the most App Store search/browse results. High impressions but low CTR may indicate a visibility vs. relevance issue. |

> **Note:** Low-volume apps (fewer than the minimum page-view threshold) are excluded from CVR rankings to avoid reading noise into small samples.

---

### App Table (per-app breakdown)

Each row represents one app for the reporting period.

| Column | Meaning |
|--------|---------|
| Impressions | Number of times the app appeared in App Store search results, browse placements, or product page referrals. |
| Product Page Views | Number of times users tapped through to the app's product page. |
| Downloads | Number of users who downloaded the app after viewing the product page. |
| CVR | Conversion Rate: Downloads ÷ Product Page Views. A high CVR means the product page is persuasive. |
| DL vs prev | Change in downloads compared to the previous reporting period (+N = improved, -N = declined). |
| CVR vs prev | Change in conversion rate in percentage points (e.g., -0.20pt means CVR dropped 0.2 percentage points). |

---

### Search Winners

Apps ranked by downloads from App Store Search impressions specifically. This shows which apps are performing well in search-driven discovery.

| Field | Meaning |
|-------|---------|
| impressions | Number of times the app appeared in search results. |
| page views | Number of times users tapped from a search result to the product page. |
| CTR | Click-through Rate: page views ÷ impressions. Measures how often the app is tapped when shown in search. |
| downloads | Number of downloads originating from search. |

---

### Emerging Apps

Apps whose key metrics (impressions, page views, or downloads) are growing week-over-week. The delta columns show the absolute change in each metric.

> **"Emerging" definition:** Apps with positive delta values in impressions and downloads, indicating upward momentum. Low-volume apps are excluded.

---

### Search Exposure, No Downloads

Apps with meaningful impressions but zero downloads. These are potential "leaky funnel" candidates — users see the app but something (relevance, pricing, screenshots, reviews) stops them from downloading.

---

### Improvement Candidates

Apps with at least some traffic (page views ≥ minimum threshold) but CVR below a target threshold (default 10%). The "Issue" column summarizes the observed pattern.

This is the most actionable section for product and marketing decisions: identify which apps have an audience but are failing to convert.

---

### Source Type Breakdown

Aggregates metrics by traffic source — App Store Search vs. App Store Browse vs. product page referrals. Useful for understanding where the audience comes from and which channel drives the most downloads.

| Source Type | Description |
|-------------|-------------|
| Search impressions total | All impressions from App Store Search. |
| Browse impressions total | All impressions from App Store Browse (tab browsing, featured sections, etc.). |
| Total page views | Sum of all product page views across all sources. |
| Per-app breakdown | Each row shows the downloads and CVR for a specific (app, source type) combination. |

---

## Generating a Local Sample

```bash
python -m src.cli --db data/sample.sqlite3 daily --mock --print
```

The generated SQLite database and report files are ignored by `.gitignore` and are safe to delete.
